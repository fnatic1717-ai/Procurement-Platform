# Phase 2A API

All routes require an authenticated active tenant membership, correlation ID propagation, and the documented permission. Decimal inputs are JSON strings. Mutation versions are positive integers; stale versions return conflict errors. Submit and approval bodies require a non-empty `idempotencyKey`. Rejection and return require a comment.

OpenAPI is generated by NestJS at `/docs`. The route groups are Purchase Requests, Approvals, Procurement Intake, and Approval Policies. Tenant IDs, requester identity, acting approver identity, and assigning actor are never read from a request body. Buyer IDs and configured named approvers are validated through composite tenant membership foreign keys and service checks.

Merge-readiness API additions include request-item update, request cancellation, approval detail, intake detail, approval-policy detail/update/status, validated list and queue query DTOs, and UUID path validation. Idempotency keys are scoped by tenant, actor, operation, object, and key; reuse with a different payload returns conflict. Role approval authorization is resolved from persisted memberships, role assignments, role permissions, and the step's required permission.
